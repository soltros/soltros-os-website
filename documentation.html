SoltrOS - Desktop Edition

A gaming-optimized, rolling immutable Linux distribution based on Fedora releases, featuring MacBook hardware support, gaming performance enhancements, the KDE Plasma desktop environment, and developer-friendly tools.

Inspired by VeneOS - bringing together the best of gaming and productivity.

Updates land weekly as images — no big version jumps or upgrade headaches. One week you might be on Fedora 42, the next it's 43, and it just works. Pick your pace: LTS follows Fedora's stable releases; Unstable tracks upcoming features from alphas and betas (with guardrails).

Quick Start

🎮 Gaming Ready

Optimized kernel with gaming enhancements, GameMode, MangoHud, and controller support out of the box.

🔒 Immutable & Stable

Atomic updates with instant rollback. Your system never breaks from updates.

💻 MacBook Optimized

Thermal management with thermald & mbpfan, Apple SMC support via applesmc kernel module.

📦 Five Package Managers

RPM-OSTree, Flatpak, Distrobox, Nix, and Homebrew for maximum flexibility.

Key Features

  • Base Image: quay.io/fedora-ostree-desktops/base-atomic (Cosmic uses cosmic-atomic upstream)
  • Desktop: KDE Plasma with SDDM (Gnome and COSMIC variants available)
  • Package Managers: 5 complementary systems (RPM-OSTree, Flatpak, Distrobox, Nix, Homebrew)
  • Pre-installed Apps: 40+ Flatpak applications
  • Shell: Zsh as default with modern tooling
  • Helper Script: helper.sh for easy system management
  • Update Model: Rolling release with atomic updates

Installation

There are two primary methods to install SoltrOS:

Method 1: ISO Installation (Recommended)

Download the official ISO, flash it to a USB device, and install it to your computer.

Bleeding-edge edition: Latest packages from Rawhide - Download Unstable ISO

Stable/LTS edition: Pinned to 6-month Fedora cycle - Download LTS ISO

Once SoltrOS is installed, it rolls updates forever. No need to swap to new ISOs every 6 months.

Creating a Bootable USB

Use a tool like Balena Etcher, Rufus (Windows), or dd (Linux) to flash the ISO to a USB drive.

# Linux example using dd
sudo dd if=soltros-os-latest.iso of=/dev/sdX bs=4M status=progress
sudo sync

Method 2: Bootc Switch from Existing System

If you're already running Fedora Silverblue, Kinoite, Bazzite, or Bluefin, you can switch directly to SoltrOS:

Switch to Unstable/KDE (main)

sudo bootc switch ghcr.io/soltros/soltros-os:latest
sudo systemctl reboot

Switch to LTS/KDE

sudo bootc switch ghcr.io/soltros/soltros-os_lts:latest
sudo systemctl reboot

Switch to Unstable COSMIC

sudo bootc switch ghcr.io/soltros/soltros-unstable_cosmic:latest
sudo systemctl reboot

Switch to LTS COSMIC

sudo bootc switch ghcr.io/soltros/soltros-lts_cosmic:latest
sudo systemctl reboot

Switch to Unstable Gnome

sudo bootc switch ghcr.io/soltros/soltros-os-unstable_gnome:latest
sudo systemctl reboot

Switch to LTS Gnome

sudo bootc switch ghcr.io/soltros/soltros-os-lts_gnome:latest
sudo systemctl reboot

Important Notice: Policy.json Fix

If you experience bootc issues due to policy.json problems, run these commands:

# Download the public key
sudo mkdir -p /etc/pki/containers
sudo curl -L https://github.com/soltros/Soltros-OS/raw/main/soltros.pub -o /etc/pki/containers/soltros.pub

# Download the secure policy
sudo mkdir -p /etc/containers
sudo curl -L https://github.com/soltros/Soltros-OS/raw/main/resources/policy.json -o /etc/containers/policy.json

# Verify permissions
sudo chmod 644 /etc/pki/containers/soltros.pub
sudo chmod 644 /etc/containers/policy.json

First Boot Setup

After installing SoltrOS, follow these steps to complete your setup:

1. Install Flatpak Applications

Install all pre-configured Flatpak applications (40+ apps including Steam, Discord, VLC, LibreOffice, and more):

sh /usr/share/soltros/bin/helper.sh install-flatpaks

2. Setup Development Environment

Configure shell with modern plugins and aliases:

sh /usr/share/soltros/bin/helper.sh setup-cli

3. Configure Git (Optional)

Setup Git with SSH signing support:

sh /usr/share/soltros/bin/helper.sh setup-git

4. Setup Development Containers (Optional)

Create Ubuntu and Arch containers for development:

sh /usr/share/soltros/bin/helper.sh setup-distrobox

5. Install Nix Package Manager (Optional)

Add access to 80,000+ packages via Nix:

sh /usr/share/soltros/bin/helper.sh install-nix

6. Install Homebrew (Optional)

Add macOS-style package manager for additional tools:

sh /usr/share/soltros/bin/helper.sh install-homebrew

7. Switch to Zsh (Optional)

Change your default shell from Bash to Zsh:

sh /usr/share/soltros/bin/helper.sh change-to-zsh

Complete Setup Script

Run all essential setup steps at once:

# Full setup
sh /usr/share/soltros/bin/helper.sh install-flatpaks
sh /usr/share/soltros/bin/helper.sh setup-cli
sh /usr/share/soltros/bin/helper.sh setup-distrobox
sh /usr/share/soltros/bin/helper.sh install-nix
sh /usr/share/soltros/bin/helper.sh change-to-zsh

MacBook Support

SoltrOS includes specialized optimizations for MacBook hardware:

Thermal Management

Aggressive thermal management for humidity resistance and optimal performance:

# From /etc/mbpfan.conf
[general]
low_temp = 58        # Start ramping fans earlier
high_temp = 62       # Aggressive ramp-up
max_temp = 78        # Full speed earlier than default
polling_interval = 1 # Check every second

Why These Settings?

  • Low Temperature (58°C): Fans start sooner than default to prevent heat buildup
  • High Temperature (62°C): More aggressive ramp-up for better cooling
  • Max Temperature (78°C): Full fan speed earlier to protect components
  • Polling Interval: 1-second checks for responsive thermal control

Hardware Integration

  • Apple SMC Support: Native sensor access via applesmc kernel module
  • Fan Control: Responsive thermal management with mbpfan
  • Hardware Monitoring: Complete sensor access with lm_sensors
  • Thermal Daemon: Additional thermal management with thermald

Automatic Module Loading

The applesmc module is automatically loaded at boot:

# From /etc/modules-load.d/applesmc.conf
applesmc

Verify MacBook Support

Check that thermal management is working:

# Check mbpfan status
systemctl status mbpfan

# Check thermald status
systemctl status thermald

# View sensor data
sensors

# Check if applesmc is loaded
lsmod | grep applesmc
