Introduction to SoltrOS

SoltrOS is an immutable Linux desktop distribution built on Fedora Core OS, designed for gamers and developers who want a stable, rolling-release system that just works.

What is SoltrOS?

SoltrOS combines the stability of immutable operating systems with the flexibility of multiple package management systems. Built on the quay.io/fedora-ostree-desktops/base-atomic image, it delivers a gaming-optimized desktop experience with comprehensive developer tools.

Immutable by Design: Your base system is read-only and updated atomically. If an update fails or causes issues, you can instantly roll back to the previous working state. No more broken systems from partial updates.

Core Philosophy

SoltrOS is built around three core principles:

🔒 Stability First

Immutable base system with atomic updates means your OS never breaks. Every update is applied as a complete unit, and rollbacks are instant.

🎮 Gaming Ready

Optimized kernel, pre-configured gaming tools, controller support, and performance tuning out of the box. MacBook thermal management included.

📦 Maximum Flexibility

Five package managers (RPM-OSTree, Flatpak, Distrobox, Nix, Homebrew) give you access to virtually any software while maintaining system integrity.

How Updates Work

Unlike traditional Linux distributions that require major version upgrades every 6 months, SoltrOS rolls forward continuously:

  • Weekly Image Updates: New system images are built and delivered weekly
  • No Version Jumps: One week you're on Fedora 42, the next week it's 43—seamlessly
  • Two Update Tracks: Choose LTS (stable releases) or Unstable (bleeding edge features)
  • Atomic Application: Updates apply completely or not at all—no broken systems
  • Instant Rollback: Don't like an update? Reboot to the previous version in seconds

Who is SoltrOS For?

Gamers

MacBook Users

  • Need responsive fan control for gaming and heavy workloads
  • Appreciate humidity-resistant thermal configurations

Why Waterfox Instead of Firefox?

SoltrOS ships with Waterfox as the default browser instead of Firefox. This is a deliberate choice based on privacy and user autonomy principles:

🔒 Privacy First

Waterfox removes Mozilla's telemetry, data collection, and tracking systems. No usage data is sent back to Mozilla or third parties by default.

🚫 No Monetization Schemes

Waterfox doesn't include Mozilla's controversial monetization features like sponsored shortcuts, "recommended" content, or paid search partnerships that compromise your privacy.

⚙️ User Control

You maintain full control over your browsing experience without being opted into experimental features, A/B testing, or "suggested" content you didn't ask for.

🔧 Firefox Compatible

Waterfox is based on Firefox ESR and maintains full compatibility with Firefox extensions, configurations, and workflows—just without the privacy concerns.

Specific Issues with Modern Firefox

  • Telemetry by Default: Firefox collects extensive usage data, even in "private" mode
  • Sponsored Content: New tab page shows sponsored shortcuts and "recommended by Pocket" articles
  • Mozilla VPN Promotions: Constant upselling of Mozilla's paid services
  • Default Search Deals: Revenue-sharing agreements with search engines that affect default settings
  • Opt-Out Rather Than Opt-In: Privacy-invasive features are enabled by default, requiring users to manually disable them
  • Firefox Suggest: Sends your search queries to Mozilla's servers for "suggestions"
  • Experiments: Users are automatically enrolled in A/B testing without explicit consent

Waterfox Philosophy

Waterfox returns to what a browser should be: a tool that works for you, not a platform that monetizes your data. It provides:

  • Clean, ad-free new tab page
  • No telemetry or data collection
  • No sponsored content or recommendations
  • No service upselling or promotions
  • Full Firefox extension compatibility
  • Regular security updates from Firefox ESR base

Want Firefox anyway? You're free to install Firefox via Flatpak if you prefer: flatpak install flathub org.mozilla.firefox. SoltrOS doesn't lock you in—we just provide privacy-respecting defaults.

Installation

SoltrOS uses custom bootable ISO images built on Fedora's immutable base. The installation process is identical to installing any Fedora distribution, but bootc deploys SoltrOS images instead of the standard Fedora Atomic ones.

Installation Methods

Method 1: ISO Installation (Recommended)

The primary installation method uses SoltrOS ISO images. These are complete, bootable installers that provide the familiar Fedora Anaconda installation experience.

How it works: When you boot the SoltrOS ISO, you'll see the standard Fedora installer (Anaconda). Behind the scenes, bootc is configured to deploy SoltrOS container images instead of the default Fedora Atomic images. The installation process looks and feels exactly like installing Fedora—because it is Fedora, just with SoltrOS pre-configured.

Step-by-Step Installation

Step 1: Create Bootable USB

After downloading your chosen ISO, create a bootable USB drive. You can use any standard tool:

On Linux:

# Using dd (replace /dev/sdX with your USB device)
sudo dd if=soltros-os-latest.iso of=/dev/sdX bs=4M status=progress
sudo sync

# Or use a GUI tool like GNOME Disks or KDE ISO Image Writer

On Windows:

  • Use Rufus
  • Or Balena Etcher

On macOS:

  • Use Balena Etcher
  • Or use the dd command in Terminal (similar to Linux)

Step 2: Boot from USB

  1. Insert the USB drive into your computer
  2. Restart your computer
  3. Enter your BIOS/UEFI boot menu (usually F2, F12, Del, or Esc during startup)
  4. Select the USB drive as the boot device
  5. The SoltrOS installer will load

Step 3: Installation Process

The SoltrOS installer is the standard Fedora Anaconda installer. If you've installed Fedora before, this will be familiar:

  1. Language Selection: Choose your language and keyboard layout
  2. Installation Destination: Select the disk where SoltrOS will be installed
    • You can use automatic partitioning (recommended for most users)
    • Or configure custom partitioning if needed
    • Minimum 20GB recommended, 40GB+ for comfortable use
  3. Network Configuration: Configure your network connection (optional during install, can be done later)
  4. Root Password: Set a root password (optional but recommended)
  5. User Creation: Create your user account
    • Choose a username
    • Set a password
    • Optionally make the user an administrator
  6. Begin Installation: Click "Begin Installation" and wait for the process to complete

Behind the Scenes: While the installer looks like standard Fedora, bootc is deploying the SoltrOS container image (ghcr.io/soltros/soltros-os:latest or your chosen variant) instead of the default Fedora Atomic image. This is what makes SoltrOS different from vanilla Fedora.

Step 4: First Boot

After installation completes:

  1. Remove the USB drive
  2. Reboot your computer
  3. You'll boot into SoltrOS with your chosen desktop environment
  4. Log in with the user account you created

Continue to the First Boot Setup page to configure your new SoltrOS installation.

Method 2: Bootc Switch (Advanced)

If you're already running a Fedora Atomic system (Silverblue, Kinoite, etc.), you can switch to SoltrOS without reinstalling:

Warning: This method is for advanced users familiar with rpm-ostree and bootc. Make sure you have backups before switching.

Switch to Unstable/KDE (main)

sudo bootc switch ghcr.io/soltros/soltros-os:latest
sudo systemctl reboot

Switch to LTS/KDE

sudo bootc switch ghcr.io/soltros/soltros-os_lts:latest
sudo systemctl reboot

Switch to Unstable COSMIC

sudo bootc switch ghcr.io/soltros/soltros-unstable_cosmic:latest
sudo systemctl reboot

Switch to LTS COSMIC

sudo bootc switch ghcr.io/soltros/soltros-lts_cosmic:latest
sudo systemctl reboot

Switch to Unstable Gnome

sudo bootc switch ghcr.io/soltros/soltros-os-unstable_gnome:latest
sudo systemctl reboot

Switch to LTS Gnome

sudo bootc switch ghcr.io/soltros/soltros-os-lts_gnome:latest
sudo systemctl reboot

Troubleshooting: Policy.json Fix

If you experience bootc issues related to container signature verification, you may need to manually configure the signing policy:

Symptom: Bootc fails with signature verification errors or policy.json issues.

Solution: Run these commands to install the correct signing keys and policy:

# Download the public key
sudo mkdir -p /etc/pki/containers
sudo curl -L https://github.com/soltros/Soltros-OS/raw/main/soltros.pub \
  -o /etc/pki/containers/soltros.pub

# Download the secure policy
sudo mkdir -p /etc/containers
sudo curl -L https://github.com/soltros/Soltros-OS/raw/main/resources/policy.json \
  -o /etc/containers/policy.json

# Verify permissions
sudo chmod 644 /etc/pki/containers/soltros.pub
sudo chmod 644 /etc/containers/policy.json

After running these commands, try your bootc switch or update again.

Verification

After installation and first boot, verify your SoltrOS installation:

# Check OS version
cat /etc/os-release

# Check bootc status
sudo bootc status

# List rpm-ostree deployments
rpm-ostree status

You should see "SoltrOS" in the OS release information and your chosen image (soltros-os, soltros-os_lts, etc.) in the bootc/rpm-ostree status.

First Boot Setup

After installing SoltrOS and logging in for the first time, you'll want to complete these setup steps to get the most out of your system. SoltrOS includes a powerful helper script that automates most of the configuration.

The Helper Script

SoltrOS includes helper.sh, a comprehensive management tool located at /usr/share/soltros/bin/helper.sh. All commands in this guide use this script.

Running helper commands: All helper commands follow this format:

sh /usr/share/soltros/bin/helper.sh <command>

For convenience, you can create an alias in your shell:

echo "alias helper='sh /usr/share/soltros/bin/helper.sh'" >> ~/.bashrc
source ~/.bashrc

# Now you can just run:
helper <command>

Step 2: Install Homebrew Package Manager

Homebrew provides access to thousands of additional packages not available in other package managers. Install it with:

sh /usr/share/soltros/bin/helper.sh install-homebrew

After installation, you can use Homebrew to install packages:

brew install <package-name>

Step 3: Install Nix Package Manager

Nix provides access to 80,000+ packages with declarative configuration. SoltrOS uses the Determinate Systems Nix installer, which is pre-configured for RPM-OSTree compatibility:

sh /usr/share/soltros/bin/helper.sh install-nix

What happens: This runs the Determinate Systems Nix installer in RPM-OSTree mode, which ensures Nix works correctly on SoltrOS's immutable filesystem. The installer is pre-downloaded to /nix/determinate-nix-installer.sh.

Step 4: Setup Nixmanager (Nix Frontend)

After installing Nix, set up nixmanager—a custom frontend script that makes managing Nix packages much easier through a bundled flake configuration:

sh /usr/share/soltros/bin/helper.sh setup-nixmanager

Nixmanager provides simple commands for Nix package management:

# Install a package
nixmanager install firefox

# Remove a package
nixmanager remove firefox

# Update all packages
nixmanager update

# Search for packages
nixmanager search <package-name>

If You're Using Bash Instead of Zsh

SoltrOS defaults to Zsh, but if you're using Bash, add the nixmanager alias:

sh /usr/share/soltros/bin/helper.sh add-nixmanager

This adds the nixmanager alias to your ~/.bashrc so it works in Bash shells.

MacBook Support

SoltrOS includes specialized optimizations for MacBook hardware:

Thermal Management

Aggressive thermal management for humidity resistance and optimal performance:

# From /etc/mbpfan.conf
[general]
low_temp = 58        # Start ramping fans earlier
high_temp = 62       # Aggressive ramp-up
max_temp = 78        # Full speed earlier than default
polling_interval = 1 # Check every second

Why These Settings?

  • Low Temperature (58°C): Fans start sooner than default to prevent heat buildup
  • High Temperature (62°C): More aggressive ramp-up for better cooling
  • Max Temperature (78°C): Full fan speed earlier to protect components
  • Polling Interval: 1-second checks for responsive thermal control

Hardware Integration

  • Apple SMC Support: Native sensor access via applesmc kernel module
  • Fan Control: Responsive thermal management with mbpfan
  • Hardware Monitoring: Complete sensor access with lm_sensors
  • Thermal Daemon: Additional thermal management with thermald

Automatic Module Loading

The applesmc module is automatically loaded at boot:

# From /etc/modules-load.d/applesmc.conf
applesmc

Verify MacBook Support

Check that thermal management is working:
